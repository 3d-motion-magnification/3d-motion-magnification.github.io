Using Positional Encoding as Point Embedding Function

"Position Shift" predicts a 3D displacement for the input point before positional encoding.
"Encoding Shift" predicts a phase shift within each sine wave for the input point during positional encoding.
We perform Linear Eulerian magnification by amplifying the temporal variations of the predicted shifts.
"Position Shift" leads to false motions, while "Encoding Shift" reduces such artifacts.


Using Tri-Plane as Point Embedding Function

"Linear - Tri-Plane" applies linear Eulerian magnification on tri-plane features.
"Phase - Tri-Plane" applies phase-based Eulerian magnification on tri-plane features.
"Linear - Tri-Plane" leads artifacts like clipped intensities, while "Phase - Tri-Plane" reduces such artifacts.



Varying the magnification factor.


Comparisons to video-based magnification.

"Observed" are Blender renderings from scenes with subtle object motions.
"Ground Truth" are Blender renderings where the true object motions are artificially amplified.
"Linear - Video" and "Phase - Video" are obtained by deploying 2D magnification methods on the non-magnified RGB videos rendered by NeRF.

In general, 3D methods that perform magnification in the embedding space produce fewer artifacts compared to 2D methods.
2D methods requires rendering at a fixed viewpoint and, for every new rendering, re-running video magnification on the fly.
In contrast, magnification on tri-planes is performed just once, and the motion-magnified tri-planes can then be used to render at new viewpoints.
